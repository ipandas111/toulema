01

产品定位与核心主张

一句话定位：校招通是"求职版 Power BI" — 把散落在多平台的投递记录、JD 信息、面经数据聚合到一个看板，并由 AI 自动情报化处理，让每一次投递决策都有数据支撑，而不是靠感觉。

v1.0 的定位偏向"AI 简历优化"，但简历打分/JD 匹配已经是 Huntr、Teal、Careerflow 等产品的标配功能，高度同质化。v2.0 重新定位在 信息聚合 + 情报分析：

旧定位新定位差异
简历优化工具求职作战指挥台从单点优化→全流程管理
用户主动粘贴 JDAI 主动搜索 JD + 面经从被动响应→主动情报
简历评分（已烂大街）面经质量评分 + 加权建议数据质量把关，差异化
纯前端，localStorage前后端分离，向量库功能深度提升，可扩展

// 竞品空白确认

调研了 14 款主流工具（Huntr、Teal、Jobright、JobSync、VibeHired 等），无一产品具备：添加岗位 → 自动搜索该岗位 JD + 面经 + 发展路径 → 质量评分过滤 → RAG 合成建议 的完整链路。这是真实的市场空白，尤其在中文语境（牛客、1point3acres、脉脉）下。

// MODULE C — 面经质量评分体系 独创机制

搜索结果质量参差不齐，未经过滤直接喂给 RAG 会产生噪音和错误建议。通过双层评分 + 交叉验证，确保入库数据可信。

设计原则：宁可召回率低一点，也要保证进入 RAG 的内容高度可信。分数 < 4 的内容直接丢弃，不进入向量库。

条件加成系数效果

// 综合评分公式

// 原始分（0-10）
raw_score = 0.25 × recency + 0.15 × density + 0.10 × platform_weight + 0.10 × community + 0.20 × llm_relevance + 0.15 × llm_specificity + 0.05 × llm_completeness

// 交叉验证调整
final_score = raw_score × cross_validate_coeff // 0.6 – 1.2

// 过滤阈值
if final_score < 4.0: discard() // 不进入向量库
elif final_score < 6.0: store(weight=0.5) // 降权入库
else: store(weight=1.0) // 正常权重

04

用户故事

US-01 · 情报自动化

作为一名正在投递 TI TSE 岗位的学生，我希望添加这个岗位后，系统自动为我搜集最近6个月的面经和该岗位JD，并告诉我面试通常考什么、几轮面试，这样我不需要自己花 2 小时去牛客上找资料。

验收：添加岗位后 30 秒内看到「情报搜集中」状态；3 分钟内情报面板显示面试流程、高频问题（≥5条）、岗位核心要求；来源列表可查看每条面经原文链接。
US-02 · 面经质量感知

作为用户，我希望看到每条面经旁边有一个质量评分，以及是「成功」还是「未通过」标签，这样我能快速判断哪条值得细读，哪条可以跳过。

验收：每条引用来源显示质量分（数字或进度条）和结果标签；质量分 < 4 的内容不显示；悬停可查看分数细分（时效性/具体性/来源）。
US-03 · 追问 AI 助手

查看情报简报后，我想追问"TI TSE的技术面主要考什么"，AI 应该基于已入库的面经回答，而不是瞎编。

验收：对话框支持多轮追问；每次回答附带来源引用（来源名+质量分）；如果向量库无相关内容，AI 明确告知"暂无该岗位相关面经，建议手动搜索"而不是生成幻觉内容。
US-04 · 投递数据洞察

我投了30家公司，想知道我的面试通过率，以及内推渠道是否比Boss直聘效果更好，帮助我决定下周应该重点做什么。

验收：Dashboard 页面显示各阶段数量 + 通过率；渠道对比图（可交互）；用文字总结"你的内推转化率高于平均XX%"等可操作建议。

// 情报引擎数据流（核心链路）

# 1. 用户添加岗位，触发异步任务
POST /api/jobs
  → 保存至 DB（状态: pending_intel）
  → 推送 Celery 任务: gather_intel(job_id)

# 2. 情报搜集（后台异步，不阻塞用户）
gather_intel(job_id):
  queries = [
    f"{company} {position} JD 岗位要求",
    f"{company} {position} 面经 2025 2026",
    f"{company} {position} 面试流程 几轮",
    f"{company} {position} 发展路径 晋升",
  ]
  results = [tavily.search(q, max_results=8) for q in queries]

# 3. 质量评分（规则层 + AI 层并行）
for doc in results:
  rule_score  = score_rules(doc)     # 时效性/字数/平台/互动
  ai_score    = score_llm(doc, job)  # Claude Haiku，$0.002/条
  final_score = weighted_merge(rule_score, ai_score)
  cross_coeff = cross_validate(doc, all_docs)
  doc.quality = final_score * cross_coeff

  if doc.quality < 4.0: continue     # 丢弃低质量

# 4. 向量化 + 入库
chunks = chunk_documents(filtered_docs)   # 300-500 tokens/chunk
embeddings = bge_model.encode(chunks)     # BGE-M3 本地模型，免费
chroma.add(embeddings, metadata={job_id, quality_score, source, outcome})

# 5. 生成初始情报简报
brief = claude_sonnet.chat(
  system="你是求职情报分析师...",
  user=f"基于以下面经，总结 {company} {position} 的面试情况",
  context=retrieve_top_k(job_id, k=10)
)

技术选型与理由

模块选型理由备选方案
前端框架 React + Vite 生态成熟，Vibe coding 工具支持好，拖拽库丰富 Vue 3（同等可行）
后端框架 Python FastAPI 异步支持好，与 LangChain/BGE/Chroma 生态无缝 Node.js Express（LLM 生态略差）
搜索 API Tavily Search API 专为 AI agent 设计，绕过反爬，中文支持好，1000次/月免费 SerpAPI（贵），自写爬虫（不稳定）
向量嵌入 BGE-M3（本地） 中英文双语最强嵌入模型，完全免费，无隐私顾虑 OpenAI Embeddings（$，英文偏）
向量库（dev） ChromaDB 纯 Python，零配置，本地运行，开发调试方便 FAISS（无元数据过滤）
向量库（prod） pgvector 复用 PostgreSQL，无需额外服务，Supabase 免费层够用 Pinecone（贵），Weaviate（重）
评分 LLM Claude Haiku 4.5 速度快，成本低（$0.002/条），准确率够用 GPT-4o-mini（同等，但你有 Claude）
合成 LLM Claude Sonnet 4.6 理解力强，中文输出质量高，RAG 合成用 GPT-4o
任务队列 Celery + Redis 情报搜集是耗时任务，必须异步，不阻塞 API RQ（更简单但功能少）
前端托管 Vercel 免费，CDN，GitHub 集成，一键部署 Netlify（同等）
后端托管 Railway $5/月起，支持 Python + Redis + PostgreSQL 一体 Render（免费但冷启动慢），Fly.io
